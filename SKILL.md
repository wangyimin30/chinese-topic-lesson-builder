# 语文专题分层学案三件套生成器

目标：把一个语文专题做成可直接用于作业、评改和讲评的分层学案三件套，而不是只整理题目和答案。

本文所说“专题分层学案”不是普通课堂学习单，而是承载专题作业、教师评改、课堂讲评的教学评一体化作业载体。学生版用于学生独立完成专题分层作业；教师版用于批改、诊断和反馈；课堂互动版用于展示思路、隐藏或呈现解析，并支持二次修改。

最终必须输出：

```text
{专题名称}专题学案（学生版）.docx
{专题名称}专题学案（教师版）.docx
{专题名称}专题学案（课堂互动版）.html
```

学生版、教师版只要 `.docx`，不要 `.doc`，不要把 HTML 改后缀伪装成 Word。课堂互动版只保留一个 HTML，不再拆 student/teacher HTML。

如果用户没有明确指定输出文件夹，默认输出到：

```text
C:\Users\{当前用户}\Desktop\{专题名称}专题学案成果
```

文件夹名和学案标题必须干净，不要出现制作过程词，例如 `docx拆表版`、`最终版`、`V5`、`可做题版`、`临时版`，除非用户明确要求。

## 触发后先做什么

1. 识别专题名称。用户只给题库或 Excel 时，从文件名、表格列、题目考点中推断专题名称。
2. 识别输入材料：
   - 学生真实困惑、错因、课堂反馈或教师想突破的专题难点。
   - 课程标准、教材单元、课后题、教参或“空中课堂”等课程依据。
   - Excel 题表或专题题库。
   - 用户指定的题库文件夹。
   - 旧学案、旧 HTML、旧 Word。
   - 教材课后习题、教参、练习册和答案。
3. 若用户只给专题名称，也要主动检索本地资料并生成学案，不要要求用户再提供题库。
4. 若用户没给输出目录，直接生成到桌面。

## 总体工作流

始终按照“课程内容 + 学生困惑 -> 确定专题目标 -> 资源检索 -> 素材筛选 -> 分层设计 -> 学案生成 -> 教师校验迭代”的逻辑推进。

1. **确定专题目标**：把课程要求、教材内容和学生困惑转化为可观察的专题目标。目标要能指导后续检索、分层、评价和讲评。
2. **资源检索**：联动课程标准、统编教材、教参、“空中课堂”文字材料、本地题库、原卷和答案。题库只是资源之一，不能把“建库”误当作学案制作第一步。
3. **素材筛选**：教师视角判断题目典型性、文本长度、分值结构、迁移价值和学生可完成度；AI辅助归类、去重、补全材料。
4. **分层设计**：先确定基础巩固、综合运用、迁移挑战三层任务梯度，再把题目组织进模块。分层不是多做少做，而是目标深度、支架强度、任务复杂度和评价重点的差异。
5. **学案生成**：生成学生作业版、教师评改版、课堂互动版三类材料。三类材料共享同一专题目标、分层模块和题目材料，但答案呈现和使用场景不同。
6. **校验迭代**：教师终审材料完整性、知识结构、学习进阶、答案解析和价值导向。根据课堂反馈继续修订，并把有效规则沉淀为后续专题可复用的生成规则。

## 专题目标规则

生成学案前先写清楚本专题要解决什么问题。若用户给的是学生困惑，从困惑反推专题目标；若用户给的是教材知识点，从课程内容和教材要求推导专题目标；若两者都有，要把“应当学什么”和“学生卡在哪里”合并为同一目标链。

目标建议写成 3-5 条可观察表现，例如：

```text
能区分……不同维度；
能依据……作出判断；
能结合文本证据解释……；
能通过迁移任务说明……。
```

不要只写“提升阅读能力”“发展核心素养”等空泛目标。

## 题目与原文规则

这是硬性要求：

- 如果用户明确给定 Excel 题表、专题题库或题库文件夹作为“必须使用”的材料，里面的题目必须全用，不能漏题。
- 如果用户只是要求围绕专题自动生成学案，要先检索形成候选题母版，再按专题目标、典型性、文本长度、分值结构、迁移价值和学生可完成度筛选；不要把全部检索结果机械塞进学案。
- 如果用户额外给了一个题库文件夹，并说明该文件夹就是本次专题题库，该文件夹中的每一道题都必须进入候选题清单；最终是否全进学案要服从用户要求和教师筛选逻辑。
- 生成前先建立题目清单；生成后核对题目数量。
- 一模、二模、中考阅读题必须尽量从题库中找到完整原文、划线句、题干、选项、表格、分值和答案。
- 额外题库里的每道题也要到真题文件夹、答案嵌入版、原卷或总表中补齐配套原文；不能只复制题干。
- 只要输入材料是中考/一模/二模阅读题表，即使用户没有额外说明“补原文”，也必须先到 `D:\msqz\中考题库`、题库索引、原卷或答案嵌入版中补齐对应原文，再生成学案；不得先生成只有题干和答案的版本。
- 学生必须能直接做题：不能只有“某文某题”，不能缺原文，不能缺划线内容。
- 如果确实找不到原文，要在完成报告中说明哪一题缺原文，不要悄悄省略。
- 教材课后习题不需要补整篇原文，但要放入合适模块，作为概念理解或迁移练习支架。
- 名著改写、综合运用、微写作等题，如果本身不是阅读原文，不要强行套“文章原文”格式。
- 原卷中出现的图表、插图、示意图、表格、流程图要尽量保留。无法提取图片时，在对应位置标注 `【图表见原卷】`，并在完成报告中说明。

## 学案结构

三件套都要包含：

- 顶部标题区：专题名称、班级、姓名。不要日期栏。
- 课程标准要求：浅黄框。要写成适配本专题的课堂化表述，不要空泛罗列。
- 本次学习总览：浅蓝框。说明本专题解决什么阅读/表达问题，怎么训练，最终迁移到什么。
- 核心知识讲解：浅黄区。知识点要专业、清晰、可答题迁移。
- 分模块讲解与练习：每个模块有模块标题、学习目标、知识要点、方法提醒、例题/真题/课后题。
- 学生答题区：学生版有横线、表格空格或作答行。
- 教师答案解析：教师版答案用红色，含参考答案、思路、评分说明或教学提示。
- 课堂互动版 HTML：每题有“解析”按钮，默认隐藏答案。

模块标题必须根据该专题的知识要点重组，不要照抄用户给的示例，也不要照抄 Excel 表格里混乱的分类。

## 分层设计规则

先做分层设计，再生成三类文件。不要把题目按检索顺序堆叠。

默认采用三层模块：

| 模块 | 作用 | 常见任务 | 支架 | 评价重点 |
|---|---|---|---|---|
| 基础巩固 | 概念辨析、方法入门 | 判断类型、识别标志、提取依据 | 判断步骤、关键词提示、示例句 | 概念是否准确 |
| 综合运用 | 方法迁移、完整作答 | 结合完整文本分析表达效果 | 阅读思考路径、文本证据提示、答题框架 | 证据是否充分，解释是否完整 |
| 迁移挑战 | 转换比较、开放表达 | 比较不同表达、片段改写、微写作 | 选做机制、比较维度、改写要求 | 能否比较差异并说明理由 |

分层的本质是目标深度、支架强度、任务复杂度和评价重点不同，不是“基础学生少做、优秀学生多做”。

生成模块时优先使用这样的路径：

```text
概念辨识 -> 方法建构 -> 典型训练 -> 综合迁移 -> 反思修正
```

若专题不适合三层结构，可以调整模块名称，但必须说明每个模块的学习目标、任务、支架和评价重点。

## 三类学案版本分工

三类文件共享同一专题目标、分层模块和题目材料，但用途不同：

| 类型 | 使用对象 | 内容构成 | 主要功能 | 生成要求 |
|---|---|---|---|---|
| 学生作业版 | 学生 | 完整选文、画线句/加点字、题干、分值、作答空间 | 独立完成专题分层作业 | 不出现答案和解析；按模块顺序呈现 |
| 教师评改版 | 教师 | 学生版全部内容、参考答案、解题思路、评分要点、易错提示 | 批改、诊断错因、准备反馈 | 答案与题目一一对应；评分要点便于判断问题类型 |
| 课堂互动版 | 教师与学生 | HTML页面、题目材料、学生思路展示区、解析按钮、答案与讲评要点 | 讲评、展示解析、组织二次修改 | 答案默认隐藏；通过“解析”按钮控制显示 |

生成时先保证学生作业版材料完整，再派生教师评改版和课堂互动版。不要先生成教师答案再删答案做学生版，以免残留解析或评分要点。

## 教师校验清单

输出前按三轮校验：

1. **材料完整性**：原文、段落、画线句、加点字、题干、选项、分值、答案是否齐全；阅读原文是否能脱离原卷独立作答。
2. **知识结构**：分类是否专业，是否把不同维度混为一类；教材、教参已有表述是否优先转化；是否存在套话或学生无法操作的表述。
3. **学习进阶**：模块顺序、题量、支架提示、作答空间、评价标准是否与基础巩固、综合运用、迁移挑战的梯度一致。

完成报告中要简要说明：题目数量、缺失材料、三类文件路径、仍需教师人工复核的地方。

## 知识点表格规则

知识点可以拆成多张小表，不要挤成一张混乱大表。

优先按“专业术语 + 教学常用说法 + 答题用途”组织。表头可使用：

```text
类型
识别标志
主要作用
答题提醒
```

或：

```text
类型（术语 + 传统名称）
识别标志
主要作用
答题提醒
```

分类必须专业。不要把不同维度混为一类。例如“叙述视角”专题应至少区分：

- 叙述人称：第一人称、第二人称、第三人称。
- 年龄与时间经验视角：儿童/少年视角、成人回望视角、双重视角。
- 聚焦类型与认知范围：零聚焦（全知视角）、内聚焦（人物限知视角）、外聚焦（客观视角）。
- 视角交织与转换：当时的我/现在的我、儿童视角/成人回望、第三人称正文/第二人称标题等。

生成知识点前必须做科学性校验：

- 术语是否准确，是否把不同维度混为一类。
- 教学常用名称与专业术语是否能互相对应。
- 识别标志、表达效果、答题提醒是否一一匹配。
- 是否存在过度概括、套话、学生无法操作的表述。
- 是否有遗漏的重要类型或反例。

如果教材、教参或教材课后习题中已有该知识点的讲解，必须优先引用或转化教材表述，并在对应模块中体现。不要只凭模型自建知识点。

避免学生听不懂的空泛术语。例如不要只写“形成时间厚度”，应写成：

```text
呈现两个时间层次：当时的经历与后来的理解，表现情感变化、理解深化和主题升华。
```

### 聚焦类型表述示例

用于叙述视角类专题时，采用更专业的聚焦表：

| 类型（聚焦术语 + 传统名称） | 识别标志 | 主要作用 | 答题提醒 |
|---|---|---|---|
| 零聚焦（全知视角） | 叙述者的认知范围大于任何单个人物，可自由进入不同人物、场景和时间。 | 信息完整，便于铺展复杂情节、群像关系和多线叙事；能形成宏观照应和人物对照。 | 关注叙述者是否越过单个人物的所见所知，是否能直接交代多方心理、背景或结果。 |
| 内聚焦（人物限知视角） | 叙述受某一人物的见闻、心理、理解和情感限制，读者主要跟随该人物获知信息。 | 增强真实感、代入感和悬念感；留下空白，制造误解、反差或理解延迟。 | “有限”本身就是表达效果，要分析它如何服务情节推进、人物塑造和主题含蓄呈现。 |
| 外聚焦（客观视角） | 叙述者像摄像机一样记录外部言行、环境、动作和声音，不进入人物内心，不直接解释或评论。 | 形成冷静、疏离、克制或悬念感；把判断空间留给读者，呈现人物心理和生活真相的不可完全知晓。 | 抓外部描写和叙述留白，分析读者需要怎样从言行细节中推断人物心理与主题。 |

## 原文排版规则

这是最容易出错的地方，必须严格执行：

- 阅读原文必须按原段落分行，不能堆成一整段。
- 保留原文段号、小标题、作者、注释、出处、“有删改”等信息。
- 原文标题、作者居中，题目正文按段落首行缩进。
- 真正的阅读文章正文用楷体，字号四号左右（Word 中 `w:sz=28`，HTML 中约 `19px`）。
- 只有阅读原文用楷体。综合运用题干、名著改写要求、微写作材料、教材课后题、普通题干不要用楷体。
- 划线句要保留。HTML 用 `<u>`，Word 中尽量使用下划线或明显标注。
- 表格题、图表题、流程图、材料框要保留原有结构，不要拆成普通段落。

典型边界：

```text
《钢铁是怎样炼成的》视角改写题里的题干和改写片段，不按阅读原文处理，不用楷体。
```

## Word 版要求

必须生成真正的 `.docx` OOXML 包：

- 包内路径使用 `/`，如 `word/document.xml`，不能出现 `word\document.xml`。
- 包内必须有 `[Content_Types].xml`、`_rels/.rels`、`word/document.xml`。
- 学生版不能出现参考答案、解析、评分说明。
- 教师版与学生版题序、材料、题干完全一致，但保留红色答案和解析。
- 学生版选择题给 `答：（    ）`；简答题按答案长度给 2-6 行横线；综合写作给 6-8 行。
- 文章原文楷体四号左右；非原文题干保持普通字体。
- 课程标准、学习总览、知识讲解要分行，不要挤成整段。

如果没有现成 docx 库，可直接构造 OOXML 包，但完成前必须校验包结构。

## HTML 互动版要求

HTML 是课堂互动版，不是导航页。

必须包含：

- 顶部标题卡。
- 浅黄课程标准卡。
- 浅蓝学习总览卡。
- 深蓝模块标题。
- 浅黄知识讲解区。
- 完整原文与题目。
- 学生作答区。
- 每题一个“解析”按钮，默认隐藏答案。

样式基准：

```css
.standard { background:#fff8c9; border-left:5px solid #d79b00; border-radius:14px; }
```

## 检索与匹配策略

优先级：

1. 用户给的 Excel/题库文件。
2. 用户额外给的题库文件夹。该文件夹每道题都必须纳入。
3. 用户给的旧学案、旧 HTML、旧 Word。
4. 默认素材库 `D:\msqz`，尤其是真题文件夹、中考题库、答案嵌入版和总表。
5. `references\模卷库`。
6. 教参、教材课后习题、教材知识讲解、练习册和答案。
7. 若没有题库，仅按专题名称从本地资料中抽题和自建支架。

```text

```

对 Excel 题表：

- 识别常见列：考试、区、板块、选文、题号、分值、题型、试题、答案、对应具体要求。
- 列名不同也要按内容映射。
- 若用户明确要求“全用”，每一行都要进入最终学案。
- 若 Excel 只是候选题母版，先全部纳入候选清单，再按专题目标、典型性和分层需要筛选进入最终学案，并在完成报告中说明筛选结果。

对一模二模/中考题：

- 用题库总表定位题源。
- 再到原卷、答案嵌入版、doc/docx/html/pdf 中找完整原文。
- legacy `.doc` 可尝试 UTF-16 读取；docx 直接解压 `word/document.xml`；HTML 用结构化解析；PDF 用可用的文本提取工具。

## 质量验收清单

完成前必须静态检查：

- 三件套都存在，且文件大小正常。
- 学生版、教师版是 `.docx`，不是 `.doc`。
- docx 包结构有效，路径包含 `word/document.xml` 和 `[Content_Types].xml`。
- 文件名和正文标题不含 `docx拆表版`、`最终版`、`临时` 等制作过程词。
- 已建立候选题清单，并说明最终入选题目数量。
- 若用户明确要求全用，Excel/题库题目数量与最终题目数量一致。
- 若用户额外给的题库文件夹并要求全用，该文件夹中每道题均已纳入最终学案。
- 一模二模/中考阅读题有完整原文和划线内容；缺失项已说明。
- 图表、表格、划线句已保留；无法提取图片时有 `【图表见原卷】` 占位说明。
- 教材课后习题已放入合适模块。
- 教材或教参中已有的知识点讲解已引用或转化进学案。
- 知识点表格分维度，分类专业，不混乱。
- 知识点经过科学性校验，没有把术语维度混合、没有套话化表述。
- 模块体现基础巩固、综合运用、迁移挑战等学习进阶，分层依据清楚。
- 学生版、教师版、课堂互动版共享同一目标、模块和题目材料。
- 阅读原文分段，楷体四号左右。
- 非阅读原文题干、综合运用、改写题、课后题不套楷体。
- 学生版没有答案；教师版答案为红色或显著标记。
- HTML 每题有解析按钮，默认隐藏答案。
- 桌面输出文件夹中没有旧版残留或嵌套文件夹。
